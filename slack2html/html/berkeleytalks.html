hellerstein
2020-11-17 17:51
Here's a rough attempt to summarize what we heard from Berkeley:
  1. *Serverless systems can benefit greatly from coordination-freeness.* We've done a bunch of work exploiting what we 'CALMly' know about monotonicity & coordination-freeness to build any-scale serverless systems with tunable, pretty-rich consistency. Anna is an example as a KVS, and Cloudburst offers familiar FaaS computation above it (hiding the monotonicity requirements from the programmer).
  2. *Coordination protocols have coordination-freeness inside!* Even protocols like Multipaxos benefit from the embarassing parallelism and sharding of monotone computation. While they have non-monotone components, we can whittle (whittaker?) them down and scale the rest. We're working on techniques to convert this from PhD invention to compiler optimization
  3. *Faceted programming can unlock these ideas for many programming models.* By separating concerns into facets, we want to enable a broad set of programming models for distributed computing, over a broad set of system/HW configs below. Underneath this is an Intermediate Representation (IR) currently called Spinach that intends to provide a provably-monotonic programming model and high performance Rust-based runtime.
